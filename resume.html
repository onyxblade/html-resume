Selected Projects

Rails plugin: camille (Type-safe data exchange between front-end and Rails)

Fresh Prep Inc.
  • API endpoints in Rails are traditionally untyped, so there could be discrepancies between what Rails returns and what the TypeScript front-end expects.
  • I made this gem to define type schema for Rails API endpoints, and then generate RPC functions in TypeScript to be called at front-end. Meanwhile, every response from Rails will be type-checked, so type discrepancies can be eliminated.
  • Comparing with other API typing framework, this gem is designed to maximize developer experience and squeeze every bit out of Ruby syntax and expressiveness.
  • https://github.com/onyxblade/camille

Rails plugin: associationist (A gem to define virtual associations)

Youyike Technology (Shopperplus Inc.)
  • By default, Rails associations are rigidly coupled with the underlying table structure. We had some complex queries that could not fit in a table or view, but we still wanted to abstract them as associations to benefit from Rails' syntax and preloading facility.
  • I created this gem to define virtual associations that do not need tables backing them. This greatly enhanced flexibility and allowed use-cases like Shopify's dynamic automated collection.
  • https://github.com/onyxblade/associationist
